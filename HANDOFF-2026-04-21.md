## Tester setup (before 2026-04-21 test)

### Build flag

In `.env` (or EAS build config) for the tester build:

```
EXPO_PUBLIC_DIAG_TOGGLE_ENABLED=true
```

Confirm on boot: logcat shows `[ENV CONFIG] DIAG_TOGGLE_ENABLED: true`.

Production store build: leave this unset → overlay is zero-cost, 5-tap is no-op.

### Showing the overlay

Tap the **medical icon in the red header** (Responder Home tab OR Incidents tab) **5 times within 2 seconds**. Overlay appears top-right. Tap 5 more times to hide. State persists across force-quit.

### Overlay fields

```
┌─ EMS DIAG ───────────────┐
│ Ping: 3s ago · OK        │  ← time since last ping + HTTP status
│ Acc: 14.2 m (present: ✓) │  ← accuracy value + whether key present in body
│ Dispatch: #123 en_route  │  ← id + server-reported status
│ Dist (server): 142.3 m   │  ← must tick DOWN as tester walks closer
│ ETA: 38 s                │
│ Auto-arrived: —          │  ← flips to green YES @ HH:mm:ss when fired
│ Last err: none           │
└──────────────────────────┘
```

## Test-day runbook (2026-04-21)

### ~5 min before test — SSH to iFastNet server

```bash
cd ~/ems-connect.psanguan.com

tail -F storage/logs/laravel.log \
  | grep -E "RESPONDER|DISPATCH|LIVEMAP|❌|ERROR" \
  | tee ~/emsdiag-$(date +%Y%m%d-%H%M).log
```

Leave the terminal open for the full test window. Ctrl+C to stop.

### Cedrix walks 200 m → 100 m → 50 m

At each checkpoint, screenshot the overlay. You're looking for:

1. `Dist (server)` keeps dropping.
2. `Acc: <number> (present: ✓)` — if `✗`, mobile isn't sending accuracy and auto-arrival caps at 100 m hard.
3. Around 100 m (or up to 150 m with good accuracy):
   - `Auto-arrived: YES @ HH:mm:ss` in green
   - Status pill flips `EN ROUTE` → `ARRIVED`
   - "I have arrived here" banner
   - CTA swaps to `Going to Hospital`
   - Haptic + toast fires
4. Tap `Going to Hospital` → navigates to hospital-navigation, pill flips to `TRANSPORTING`.

### Server-side tail should show (same moment)

| Line | Meaning |
|---|---|
| `[RESPONDER] 📍 Location updated` | Ping arrived, `accuracy` visible in payload |
| `[DISPATCH] 📐 Distance refreshed` | Road distance recomputed (drives the web bar) |
| `[DISPATCH] 📏 Auto-arrival near-miss` | Close but outside threshold — shows `distance_meters` + `effective_threshold_meters` |
| `[DISPATCH] 📍 Auto-arrival triggered by proximity` | **Win** — backend flipped to `arrived` |
| `[RESPONDER] ❌ Failed to update location` | **Investigate** — send me the message |
| `[LIVEMAP] Failed to fetch route history` | Separate issue, ignore |

## Common-failure triage

| Symptom | Likely cause | Fix |
|---|---|---|
| Distance doesn't tick down | Pings failing OR `active_dispatch` null | Overlay `Ping` row shows status; if 422 → off-duty, if 500 → server, if NETWORK → signal |
| Auto-arrival won't fire below 150 m | `accuracyPresent: ✗` in overlay | Mobile build doesn't have accuracy patch — rebuild |
| Auto-arrival won't fire even with `accuracyPresent: ✓` | GPS accuracy too tight (backend limit = 100 m + accuracy) | Server: bump `AUTO_ARRIVAL_THRESHOLD_METERS=120` env var |
| CTA doesn't swap despite server flipping status | Polling not returning status (10 s interval) OR mobile build is stale | Force-quit + reopen; overlay `Dispatch` row should show `arrived` |
| Second pin invisible on map for same-coord reports | Old build (no jitter patch) | Rebuild — `applyJitterForOverlaps` added in today's patch |

```
EMSDIAG {"event":"location_ping","status":"OK","accuracy":14.2,"accuracyPresent":true,"httpStatus":200,"responseDistance":142.3,"dispatchStatus":"en_route","autoArrived":false,"ts":"..."}
EMSDIAG {"event":"auto_arrived","dispatchId":123,"arrivedAt":"..."}
EMSDIAG {"event":"status_transition","dispatchId":123,"from":"en_route","to":"arrived","trigger":"server"}
EMSDIAG {"event":"button_rendered","screen":"route-map","label":"Going to Hospital","dispatchStatus":"arrived","dispatchId":123}
EMSDIAG {"event":"error","kind":"http_500","path":"/api/responder/location","message":"..."}
EMSDIAG {"event":"overlay_toggled","visible":true}
```

One line per event, prefixed `EMSDIAG`. Greppable in both `adb logcat` and Xcode Console.

## Not done yet / nice-to-haves for later

- Auto-resume from 422 pause currently only triggers when polling reveals a non-completed dispatch. A manual "I'm on duty" toggle in the UI would be cleaner.
- The jitter ring is deterministic but not clustered — if >5 pins stack, the ring gets crowded. Revisit with `react-native-maps-super-cluster` if that becomes a real-world problem.
- Overlay doesn't yet show GPS permission state or background-tracking active flag. Add if testers ask.
- `EMSDIAG` events ship in production builds — intentional (one per 5 s ping is fine). If it becomes noisy, gate on `DIAG_TOGGLE_ENABLED`.
